# 🎨 프론트엔드 아키텍처 및 디자인 시스템 (Korea APT Index)

본 문서는 대시보드의 일관된 사용자 경험(UX)과 확장성 높은 코드 구조를 유지하기 위한 프론트엔드 설계 명세서입니다.

## 1. 🌈 색상 팔레트 및 디자인 시스템 (Design Tokens)

부동산과 금융 데이터는 **'신뢰감'**과 **'직관성'**이 생명입니다.
한국의 시장 통념(오르면 빨간색, 내리면 파란색)을 반영하여 디자인 시스템을 구축합니다.

* **Primary (브랜드 메인 컬러 - 신뢰/전문성):**
  * Indigo (`bg-indigo-600` / `#4f46e5`): 주요 버튼, 활성화된 메뉴, 메인 타이틀
  * Indigo Light (`bg-indigo-50`): 메인 배경 강조용

* **Semantic (데이터 상태 표시 컬러):**
  * **상승 (Up/Hot):** Rose (`text-rose-500` / `#f43f5e`) - 지수 상승, 급등 지역 표시
  * **하락 (Down/Cold):** Blue (`text-blue-500` / `#3b82f6`) - 지수 하락, 침체 지역 표시

* **Neutral (배경 및 텍스트 - 가독성):**
  * 배경(Background): Slate-50 (`#f8fafc`) - 눈이 편안한 아주 옅은 회색
  * 카드(Surface): White (`#ffffff`) - 데이터가 올라가는 패널 (Shadow 적용)
  * 기본 텍스트: Slate-800 (`#1e293b`)
  * 보조 텍스트: Slate-500 (`#64748b`)

## 2. 📐 UI/UX 레이아웃 구조 (Layout)

전문적인 B2B/대시보드 SaaS 형태의 **[사이드바 + 메인 컨텐츠]** 구조를 채택합니다.

+----------------+---------------------------------------------------+
|                | [Header] 글로벌 필터 (예: 기준 연월: 2026년 03월) |
|   [Sidebar]    +---------------------------------------------------+
|  (네비게이션)  |                                                   |
|                |                   [Main Content]                  |
| - 요약 대시보드|                                                   |
| - 지역별 트렌드|      +---------------+     +---------------+      |
| - 전국 지도 뷰 |      | KPI Card 1    |     | KPI Card 2    |      |
| - 시장 랭킹    |      +---------------+     +---------------+      |
|                |      +-------------------------------------+      |
|                |      |           Chart Area                |      |
|                |      +-------------------------------------+      |
+----------------+---------------------------------------------------+

* **Sidebar (LNB):** 메뉴 이동이 잦은 대시보드 특성상 좌측 고정 사이드바로 구성.
* **Header (GNB):** 현재 보고 있는 데이터의 '전역 기준 시간(Target Month)' 등을 설정하는 곳.
* **Dashboard Cards:** 숫자로 된 요약 정보(예: "최고 상승 지역")를 상단에 배치하여 인사이트를 즉시 제공.

## 3. 📂 디렉토리 구조 (Next.js App Router 정석)

컴포넌트의 재사용성을 극대화하기 위해 역할을 명확히 분리합니다.

frontend/src/
  ├── app/                  # 페이지 라우팅 및 전역 레이아웃
  │   ├── layout.tsx        # 글로벌 레이아웃 (사이드바, 헤더 포함)
  │   ├── page.tsx          # 메인 요약 대시보드 (Home)
  │   ├── trend/page.tsx    # 지역별 트렌드 분석 페이지
  │   └── map/page.tsx      # 지도 시각화 페이지
  ├── components/           # 재사용 가능한 UI 컴포넌트
  │   ├── layout/           # Sidebar, Header, PageContainer
  │   ├── ui/               # Card, Button, SelectBox (디자인 토큰 적용)
  │   └── charts/           # BarChart, LineChart (Recharts 래핑)
  ├── lib/                  # 유틸리티 함수 및 API 클라이언트
  │   ├── api.ts            # Axios 인스턴스 세팅
  │   └── utils.ts          # 숫자 콤마, 날짜 포맷 함수 등
  └── types/                # TypeScript 인터페이스 정의
      └── index.ts          # Region, TrendData 등

## 4. 💡 페이지별 UX 기획 (User Journey)

1. **메인 대시보드 (`/`) - "시장 요약"**
   * **목적:** 사이트 접속 시 현재 부동산 시장의 '온도'를 한눈에 파악.
   * **구성:** 전국 평균 지수 KPI 카드, 상승/하락 TOP 5 막대 그래프.

2. **트렌드 분석 (`/trend`) - "딥 다이브(Deep Dive)"**
   * **목적:** 특정 지역을 선택하여 과거부터 현재까지의 흐름을 분석.
   * **구성:** 지역 선택 드롭다운, 시계열 꺾은선 그래프, 전월 대비 증감률 텍스트.

3. **시장 랭킹 (`/ranking`) - "비교 분석"**
   * **목적:** 전체 지역을 테이블 형태로 나열하고 오름차순/내림차순 정렬하여 탐색.
